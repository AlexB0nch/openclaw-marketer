# TG Scout Agent — Skills

## Skill 1: Поиск каналов

**Описание:** Поиск публичных Telegram-каналов по ключевым словам продукта через Telethon.

**Входные данные:**
- `keywords: list[str]` — ключевые слова для поиска
- `min_subscribers: int` — минимальный порог подписчиков (по умолчанию 1000)
- `min_er: float` — минимальный ER (по умолчанию 0.01 = 1%)

**Выходные данные:**
- `List[ChannelInfo]` — список найденных каналов с метриками

**Пример:**
```python
channels = await scout.search_channels(
    keywords=["AI помощник", "B2B инструменты"],
    min_subscribers=5000,
    min_er=0.03,
)
# → [ChannelInfo(username="aitools_ru", title="AI Tools RU", subscriber_count=12500, er=0.072, ...)]
```

## Skill 2: Обогащение метриками TGStat

**Описание:** Дополнение данных каналов метриками из TGStat API (точный ER, рост, категория).

**Входные данные:**
- `channels: List[ChannelInfo]` — список каналов после поиска

**Выходные данные:**
- `List[ChannelInfo]` — обогащённый список. Если `TGSTAT_API_KEY` пуст — возвращает исходный список без изменений.

**Пример:**
```python
enriched = await tgstat.enrich_channels(channels)
# Если api_key="" → возвращает channels без изменений (тихо)
# Иначе → обновляет subscriber_count, avg_views, er, source="tgstat"
```

## Skill 3: Оценка релевантности

**Описание:** Оценка пригодности канала для продвижения продукта по 100-балльной шкале.

**Входные данные:**
- `channel: ChannelInfo` — данные канала
- `product: str` — идентификатор продукта (`ai_assistant` | `ai_trainer` | `ai_news`)

**Выходные данные:**
- `RelevanceScore` — оценка с разбивкой по компонентам

**Формула:**
| Компонент | Диапазон | Описание |
|---|---|---|
| `size_score` | 0–25 | Log10-шкала: 0 при <1K, 25 при >100K подписчиков |
| `er_score` | 0–25 | 0 при ER<1%, 25 при ER≥10% |
| `topic_score` | 0–25 | Совпадение ключевых слов |
| `semantic_score` | 0–25 | Оценка Claude Haiku по описанию |

**Пример:**
```python
score = await scorer.score_channel(channel, "ai_assistant")
# → RelevanceScore(channel_username="aitools_ru", score=87,
#     breakdown={"size_score":20, "er_score":18, "topic_score":25, "semantic_score":24})
```

## Skill 4: Генерация питча

**Описание:** Создание персонализированных питч-текстов трёх форматов через Claude API.

**Входные данные:**
- `channel: ChannelInfo` — данные канала
- `product: str` — продукт для продвижения
- `relevance_score: RelevanceScore` — оценка релевантности

**Выходные данные:**
- `PitchDraft` — три версии питча на русском языке, статус `pending_approval`

**Форматы:**
- `pitch_short` — ДМ-сообщение (≤200 символов), неформальный тон
- `pitch_medium` — Email-питч (100–300 слов), профессиональный тон
- `pitch_long` — Полное КП (300–600 слов) с заголовками

**Пример:**
```python
draft = await pitcher.generate_pitch(channel, "ai_trainer", score)
# → PitchDraft(
#     pitch_short="Привет! У вас классный фитнес-канал. Наш AI-тренер...",
#     pitch_medium="Добрый день! Мы заметили ваш канал о тренировках...",
#     pitch_long="## Коммерческое предложение\n...",
#     status="pending_approval"
# )
```

## Skill 5: Мониторинг упоминаний

**Описание:** Непрерывное прослушивание публичных каналов и групп Telegram для обнаружения упоминаний продуктов.

**Входные данные:**
- Ключевые слова из `MONITOR_KEYWORDS` env-переменной (через запятую)

**Выходные данные:**
- Telegram-уведомление администратору при каждом новом упоминании с кнопками:
  - 👀 Посмотреть — ссылка на сообщение
  - 💬 Ответить — триггер для ответа
  - 🔕 Игнорировать — снятие уведомления

**Дедупликация:** одно и то же сообщение (по `message_id`) не вызывает повторного уведомления.

**Пример уведомления:**
```
🔔 Упоминание: "AI помощник"
📢 Канал: @biztools_ru
💬 Коллеги, кто пользовался AI помощником для переписки? Ищу что-то для…
[👀 Посмотреть] [💬 Ответить] [🔕 Игнорировать]
```

## Skill 6: Еженедельный дайджест

**Описание:** Формирование и отправка топ-10 каналов недели администратору для одобрения питчей.

**Расписание:** каждый понедельник в 09:00 МСК (автоматически) или POST `/api/v1/scout/digest/send`.

**Входные данные:** нет (берёт данные из БД)

**Выходные данные:** серия Telegram-сообщений в чат администратора

**Каждая карточка содержит:**
- Название канала, подписчики, ER, общий счёт
- Тематику канала
- Превью короткого питча
- Кнопки: ✅ Отправить питч | 👀 Показать полный | ❌ Пропустить
